Accessibility Statement for DevPrompt Studio

Last Updated: January 2025

Our Commitment

DevPrompt Studio is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and apply relevant accessibility standards.

Conformance Status

DevPrompt Studio conforms to WCAG 2.1 Level AA standards. This means our application:

  • Is perceivable - Information and UI components are presentable to users in ways they can perceive
  • Is operable - UI components and navigation are operable by all users
  • Is understandable - Information and operation of the UI are understandable
  • Is robust - Content can be interpreted by a wide variety of user agents, including assistive technologies

Accessibility Features

Keyboard Navigation

  • Full keyboard support for all interactive elements
  • Skip navigation links to bypass repetitive content
  • Logical tab order throughout the application
  • Keyboard shortcuts: Ctrl+E (enhance), Ctrl+S (save), Ctrl+K (focus input)

Screen Reader Support

  • Semantic HTML structure with proper heading hierarchy
  • ARIA labels and descriptions for all interactive elements
  • Live regions for dynamic content updates
  • Alternative text for all images and icons

Visual Design

  • WCAG AA compliant color contrast ratios (minimum 4.5:1 for text)
  • Focus indicators on all interactive elements
  • Responsive design supporting 200% zoom without loss of functionality
  • Support for high contrast mode
  • Reduced motion support for users with vestibular disorders

Forms and Error Handling

  • Clear labels for all form inputs
  • Error messages announced to screen readers
  • Inline validation with descriptive error messages
  • Required fields clearly marked

Tested With

  • Screen Readers: NVDA, JAWS, VoiceOver
  • Browsers: Chrome, Firefox, Safari, Edge
  • Tools: axe DevTools, WAVE, Pa11y

Known Limitations

We are aware of the following accessibility limitations and are working to address them:

  • Some complex data visualizations may have limited screen reader support
  • Real-time streaming content may not be fully accessible to all assistive technologies

Feedback

We welcome your feedback on the accessibility of DevPrompt Studio. Please contact us:

  • Email: accessibility@devpromptstudio.com
  • GitHub: Report an issue

We aim to respond to accessibility feedback within 5 business days.

Technical Specifications

DevPrompt Studio relies on the following technologies:

  • HTML5
  • CSS3
  • JavaScript (React)
  • ARIA (Accessible Rich Internet Applications)

Assessment Approach

DevPrompt Studio was assessed using:

  • Automated testing with axe-core and Pa11y
  • Manual testing with keyboard navigation
  • Screen reader testing (NVDA, JAWS, VoiceOver)
  • User testing with people with disabilities

Formal Complaints

If you are not satisfied with our response to your accessibility concern, you may escalate your complaint to:

  • Your local disability rights organization
  • The Web Accessibility Initiative (WAI) at W3C
